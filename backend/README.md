# CivicPulse API

FastAPI service for CivicPulse AI.

```bash
python3 -m venv .venv
.venv/bin/python -m pip install -e ".[dev]"
.venv/bin/alembic -c alembic.ini upgrade head
.venv/bin/uvicorn app.main:app --reload
```

Health endpoints:

- `GET /health/live`
- `GET /health/ready` — verifies the expected database migration revision

Developer inspection:

- `GET /docs` — interactive OpenAPI documentation
- `GET /api/v1/meta` — supported categories, severities, statuses, and community actions

Reporting endpoints:

- `POST /api/v1/reports/analyze` — validate an image, analyze it, and create a private draft
- `GET /api/v1/reports/{draft_id}` — retrieve a reviewable draft
- `PATCH /api/v1/reports/{draft_id}` — save citizen corrections
- `POST /api/v1/reports/{draft_id}/publish` — publish once and create the initial timeline entry
- `DELETE /api/v1/reports/{draft_id}` — cancel a private draft
- `GET /api/v1/media/{key}` — serve locally stored development images

Use `AI_PROVIDER=demo` for deterministic local development or
`AI_PROVIDER=gemini` with a backend-only `GEMINI_API_KEY`. Use
`STORAGE_BACKEND=local` during development and `STORAGE_BACKEND=gcs` with
`STORAGE_BUCKET` in production. The API verifies actual JPEG, PNG, or WebP
content before storage; filenames and storage keys are generated by the server.

## PostgreSQL Tests

The default test suite verifies models, schemas, migration SQL, API behavior, privacy boundaries, and request tracing without modifying a developer database.

To run the destructive upgrade/downgrade integration test, create a dedicated PostgreSQL database whose name contains `_test`:

```bash
TEST_DATABASE_URL=postgresql+psycopg://user:password@localhost:5432/civicpulse_test \
  .venv/bin/pytest -m postgres
```

Never point `TEST_DATABASE_URL` at a development or production database.
