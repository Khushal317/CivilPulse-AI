# CivicPulse API

FastAPI service for CivicPulse AI.

```bash
python3 -m venv .venv
.venv/bin/python -m pip install -e ".[dev]"
.venv/bin/alembic -c alembic.ini upgrade head
.venv/bin/uvicorn app.main:app --reload
```

Health endpoints:

- `GET /health/live`
- `GET /health/ready` — verifies the expected database migration revision

Developer inspection:

- `GET /docs` — interactive OpenAPI documentation
- `GET /api/v1/meta` — supported categories, severities, statuses, and community actions

Reporting endpoints:

- `POST /api/v1/reports/analyze` — validate an image, analyze it, and create a private draft
- `GET /api/v1/reports/{draft_id}` — retrieve a reviewable draft
- `PATCH /api/v1/reports/{draft_id}` — save citizen corrections
- `POST /api/v1/reports/{draft_id}/publish` — publish once and create the initial timeline entry
- `DELETE /api/v1/reports/{draft_id}` — cancel a private draft
- `GET /api/v1/media/{key}` — serve locally stored development images

Public tracker:

- `GET /api/v1/issues` — paginated public issue cards
- `GET /api/v1/issues/{issue_id}` — public issue detail, timeline, counts, and viewer signals
- `POST /api/v1/issues/{issue_id}/community-actions` — idempotent anonymous community signal
- Filters: `category`, `severity`, `status`, and `location`
- Sorts: `newest`, `oldest`, `most_verified`, and `severity`
- Pagination: `page` and `page_size` with a maximum page size of 50

Administrator API:

- `POST /api/v1/admin/auth/login` — verify secret-backed credentials and create a session
- `GET /api/v1/admin/auth/session` — retrieve the current administrator and CSRF token
- `POST /api/v1/admin/auth/logout` — revoke the current session
- `GET /api/v1/admin/dashboard` — protected metrics, category breakdown, latest reports, and priority queue
- `GET /api/v1/admin/issues` — protected search, filters, and pagination
- `GET /api/v1/admin/issues/{issue_id}` — protected full issue record and authorized private contact details
- `POST /api/v1/admin/issues/{issue_id}/status` — validated status transition and public timeline entry

Generate a scrypt password hash with
`.venv/bin/python -m scripts.hash_admin_password`, then configure
`ADMIN_USERNAME`, the single-quoted `ADMIN_PASSWORD_HASH`, and a strong
`ADMIN_SESSION_SECRET`. Write requests require the session CSRF token.

Load realistic local tracker data with:

```bash
docker compose exec backend python -m scripts.seed_demo
```

Use `AI_PROVIDER=demo` for deterministic local development or
`AI_PROVIDER=gemini` with a backend-only `GEMINI_API_KEY`. Use
`STORAGE_BACKEND=local` during development and `STORAGE_BACKEND=gcs` with
`STORAGE_BUCKET` in production. The API verifies actual JPEG, PNG, or WebP
content before storage; filenames and storage keys are generated by the server.

Community actors receive an HttpOnly signed cookie. Only a SHA-256 actor hash is
stored with community actions. Configure `ANONYMOUS_ACTOR_SECRET`,
`COMMUNITY_ACTION_RATE_LIMIT`, and
`COMMUNITY_ACTION_RATE_WINDOW_MINUTES` in production.

Administrator sessions use random browser tokens in Strict SameSite, HttpOnly
cookies. Only SHA-256 token hashes are persisted. Sessions expire, can be
revoked on logout, and login attempts are rate limited.

## PostgreSQL Tests

The default test suite verifies models, schemas, migration SQL, API behavior, privacy boundaries, and request tracing without modifying a developer database.

To run the destructive upgrade/downgrade integration test, create a dedicated PostgreSQL database whose name contains `_test`:

```bash
TEST_DATABASE_URL=postgresql+psycopg://user:password@localhost:5432/civicpulse_test \
  .venv/bin/pytest -m postgres
```

Never point `TEST_DATABASE_URL` at a development or production database.
